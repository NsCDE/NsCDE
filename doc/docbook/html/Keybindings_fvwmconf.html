9.16. Keybindings.fvwmconf

This file is a kind of a wrapper. According to the value of the $[infostore.kbd_bind_set] it will read either Keybindings.cua or Keybindings.nscde1x file, as well as menu-bounded keybindings from the $NSCDE_DATADIR/defaults/Keymenu-cua.actions or $NSCDE_DATADIR/defaults/Keymenu-nscde1x.actions. This are the two sets of key bindings: "nscde1x" implements non-standard and arbitrary choice key bindings used until NsCDE 2.1, while "cua" implements IBM Common User Access rules, which are more in compliance with other environments and also CDE. According to the old CDE documentation, there are only couple of key bindings known under CDE if we condider window manager and DE, not the Motif and X resources of individual applications. In both this sets of key bindings, NsCDE tries to combine a CDE legacy with a number of new functions and shortcuts, originally not present in CDE. It is recommended to use more standard and more structured new "cua" set of key bindings. For a describtion of all key bindings, see chapter Common User Access (CUA) Key Bindings or NsCDE 1.X Key Bindings.
